\n# Milestone 3 — Messages, Communities & Real-Time Layer\n\nScope is large. To keep it shippable and reviewable, I'll deliver M3 in **3 sub-phases**, each independently testable in the preview via the `DEV_OFFLINE_MODE` localStorage layer (same pattern as M2), with the real Express+Socket.IO backend scaffolded in parallel so it's drop-in ready.\n\n---\n\n## Phase 3A — Foundations (real-time + data layer)\n\n**Backend (`/server`)**\n- `realtime/socket.ts` — Socket.IO server, JWT handshake auth, room-based broadcast.\n- Mongo models: `Conversation`, `Message`, `Community`, `Channel`, `CommunityMember`, `Notification`.\n- `/api/v1/conversations`, `/api/v1/messages`, `/api/v1/communities`, `/api/v1/communities/:id/channels`, `/api/v1/notifications` — full CRUD + pagination + Zod validation.\n- Socket events: `message:new`, `message:read`, `typing:start/stop`, `presence:online/offline`, `notification:new`, `channel:message`.\n- Role middleware: owner / admin / moderator / member.\n- Rate limiting on message + community-create endpoints.\n\n**Frontend (`/src`)**\n- `lib/socket.ts` — singleton Socket.IO client (swappable transport).\n- `lib/store/messages.ts`, `lib/store/communities.ts`, `lib/store/notifications.ts` — localStorage-backed mocks for offline mode.\n- `lib/api/messages.ts`, `lib/api/communities.ts`, `lib/api/notifications.ts` — typed clients.\n- `lib/hooks/use-conversations.ts`, `use-messages.ts`, `use-communities.ts`, `use-presence.ts`, `use-typing.ts`.\n- Types added to `lib/types.ts`.\n\n## Phase 3B — UI: DMs, Communities, Channels, Discovery\n\n**New routes**\n```text\n/messages                  ← inbox: DMs + joined community channels (split tabs)\n/messages/dm/$id           ← 1:1 conversation\n/communities               ← Explore (trending, recommended, categories, search)\n/communities/$id           ← community detail (about, channels list, members)\n/communities/$id/$channel  ← channel chat view\n/communities/new           ← creation flow\n/notifications             ← (already exists, gets wired to real data)\n```\n\n**Components (`src/components/messaging/`)**\n- `ConversationList`, `ConversationItem`, `MessageThread`, `MessageBubble`, `MessageComposer`, `TypingIndicator`, `PresenceDot`, `UnreadBadge`.\n- `CommunityCard`, `CommunityHeader`, `ChannelSidebar`, `ChannelHeader`, `ChannelList`, `MemberList`, `RoleBadge`.\n- `CreateCommunitySheet`, `JoinCommunityButton`, `PinnedResources`.\n\n**BottomNav** updated: Home · Focus · **Messages** · Rooms · Sage · Profile (Tasks moves into Home dashboard quick actions; we have 6 nav slots, kept).\n\n**Design**: dark theme, neon-yellow accents only on unread/active state, Syne for community/channel titles, DM Sans body, DM Mono for metadata. No reaction picker, no GIFs, no emoji bar — composer is plain text + (future) attach. Discord-density on desktop, mobile-first stacked layout.\n\n## Phase 3C — Sage AI + Study Room Integration\n\n**Backend AI service (`server/src/ai/`)**\n- `provider.ts` — unified `chat()`, `stream()` interface.\n- `providers/gemini.ts` — concrete Gemini provider using `GEMINI_API_KEY` env var.\n- `context-builder.ts` — pulls profile + recent tasks + sessions + (optional) community/channel context.\n- `rate-limit.ts` — per-user token bucket.\n- Routes:\n  - `POST /api/v1/sage/chat` (SSE streaming)\n  - `POST /api/v1/sage/community-summary`\n  - `POST /api/v1/sage/study-recommendations`\n  - `POST /api/v1/sage/session-assistance`\n- Persists to `SageConversation` + `SageMessage`.\n\n**Frontend Sage**\n- `/sage` page upgraded: streaming chat, suggested prompts, context chips.\n- Inline \
